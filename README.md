# DoorDash Design System
### Built with Figma MCP × Claude Code

A DoorDash-inspired React component library generated using the Figma MCP server and Claude Code — 
exploring what's possible when AI can read your design files directly.

🔗 **[Live Demo](https://saadaf-m.github.io/doordash-design-system/)**

---

## What is this?

The goal: Find an AI tool that could meaningfully help designers in their day-to-day — and demonstrate it.
I chose to explore the **Figma MCP × Claude Code** workflow to build a coded design system inspired by DoorDash's visual language.

---

## Why this workflow?

The product development cycle has changed. Designers, PMs, and researchers are increasingly expected to build out ideas themselves using AI — rather than waiting on traditional design-to-dev handoff cycles.

This workflow removes the translation layer between design and code:
- **Before:** Describe your designs to AI in text. AI generates a best guess.
- **After:** Claude reads your Figma file directly — layouts, components, tokens, and all.

---

## Tech Stack

- **Figma MCP Server** — connects Claude Code to Figma via OAuth
- **Claude Code** — agentic AI coding tool by Anthropic
- **React** — component library framework
- **Tailwind CSS** — utility-first styling
- **Node.js** — runtime environment
- **GitHub Pages** — deployment

---

## How it was built

1. Duplicated the **Untitled UI** design system from Figma Community
2. Connected Claude Code to Figma via the remote MCP server
3. Prompted Claude Code to read the Figma file and generate a DoorDash-inspired React component library
4. Deployed to GitHub Pages
<br>
The key thing to note here is that for this to work, the MCP needs to be able to read something. So I decided to start with a basic _already-made_ design system I found from Figma community.
